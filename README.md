## `// architecture_principles`

```bash
┌─[mayur@fsociety]─[~]
└──╼ $ cat principles.txt

01  |  THE BORING STACK WINS
    |  Reach for proven tools before shiny ones.
    |  Boring is maintainable. Boring ships.

02  |  DATA IS THE HARDEST PART
    |  Get the schema right before you write a single route.
    |  A bad data model poisons everything downstream.

03  |  APIs ARE CONTRACTS
    |  Break them and you break trust.
    |  Version everything. Document everything.

04  |  COMPLEXITY IS DEBT
    |  Every abstraction you add, someone has to understand later.
    |  Add it only when the pain of NOT having it is real.

05  |  OBSERVABILITY IS NOT OPTIONAL
    |  If you can't see what's happening in production,
    |  you're flying blind. Log. Measure. Alert.
```
